This documentation is structured around the operational lifecycle of a self-hosted system — not around specific software or hardware.

## Sections

### [Operations](/docs/operations/)

Backups, restore testing, monitoring, patching, and incident habits. Most outages are operational. Start here.

### [Architecture](/docs/architecture/)

Compute, storage, networking, power, and redundancy decisions — with explicit tradeoffs at homelab scale.

### [Scaling](/docs/scaling/)

When to keep workloads at home, when to move them, and how to manage growth without accumulating operational debt.

## How to use this reference

Each page is written to answer a specific operational question. Start with [Operational baselines](/docs/operations/baselines/) — the minimum bar before adding complexity.

Pages reference each other where relevant. If you follow a link and the target page is thin, that is a known gap, not an error.
